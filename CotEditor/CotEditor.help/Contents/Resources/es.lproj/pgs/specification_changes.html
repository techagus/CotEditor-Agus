Cambios importantes en CotEditor 2.0

CotEditor 2.0 incluye algunos cambios de especificación para la personalización de las versiones anteriores. Si has usado la versión heredada antes, es posible que necesites algunas migraciones manuales.

Formato de estilo de sintaxis

El formato para la definición del estilo de sintaxis ha cambiado significativamente. Los estilos personalizados por el usuario se traducirán automáticamente al nuevo formato de estilo en el primer lanzamiento de CotEditor 2. Sin embargo, recomendamos actualizar o reescribir sus estilos para ajustarse a las nuevas características para el estilo de sintaxis, como el cambio de comentarios o las nuevas variaciones de resaltado.

Puede encontrar algunos nuevos estilos de sintaxis adicionales en nuestra página Wiki.

Motor de expresión regular para la extracción de esquemas

Aquellos que tienen estilos de sintaxis personalizados en CotEditor 1.x pueden tener que arreglar su definición para extraer el menú de esquema manualmente, porque el motor de expresión regular para el análisis de esquema se ha cambiado en 2.0 de Oniguruma a la biblioteca de la ICU. Valida tus estilos de sintaxis personalizados en el editor de estilos de sintaxis y sigue los mensajes.

Los puntos típicos a arreglar son:

  • El “{” y “}” son caracteres que siempre se deben escapar con una barra invertida “\” en ICU, esto no hacía falta en Oniguruma.
  • La definición de “$&” para toda la cadena coincidente se eliminó en 2.0. Usa “$0” en su lugar para obtener el mismo efecto.

Ver “Usa la expresión regular” para toda la sintaxis de expresiones regulares.

API para AppleScript

Algunas especificaciones para el soporte de scripting se han actualizado en CotEditor 2.0. A partir de esto, es posible que algunos scripts de usuario no se ejecuten correctamente en 2.0 sin modificarlos. Ver “Script CotEditor con AppleScript” para los detalles de este cambio de especificación.

Identificador de paquete

TEl identificador del paquete para CotEditor se cambió de “com.aynimac.CotEditor” a “com.coteditor.CotEditor”. FRom este cambio, las asignaciones de archivos u otros enlaces a CotEditor (por ejemplo, editor externo) fuera de CotEditor se deben restablecer. Por favor, enlázalos de nuevo, si se han perdido los enlaces.

Para usuarios avanzados: Tenga en cuenta que los nombres de archivo para las preferencias y el nombre de dominio para el comando “defaults” en el Terminal también ha cambiado.
