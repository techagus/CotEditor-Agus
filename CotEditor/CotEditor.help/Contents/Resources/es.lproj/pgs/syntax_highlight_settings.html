Definir el resaltado de sintaxis en CotEditor

Tipos de resaltado de sintaxis

Los elementos de código que utiliza la coloración de sintaxis de CotEditor son los siguientes:

ElementoDescripciónEjemplos
Palabras clave Palabras clave en el código fuente, como palabras reservadas de lenguajes de programación. for, while, if
Comandos Funciones o comandos en el código fuente. print, filter, func()
Tipos Tipos en el código fuente o encabezados en lenguaje estructurado. int, bool, Object
Atributos Atributos en el código fuente. <html lang="ja">, @classmethod
Variables Variables en el código fuente. $foo, self
Valores Valores constantes en el código fuente. Null, True, False
Números Constantes numéricas en el código fuente. Int i = 0;
Cadenas Constantes de cadena en el código fuente. String s = "Hola Mundo"
Caracteres Constantes de caracteres en el código fuente. Char c = 'a'
Comentarios Comentarios en el código fuente. /* 〜 */, # 〜

CotEditor colorea los elementos de código en el siguiente orden: 1) palabras clave, 2) comandos, 3) tipos, 4) atributos, 5) variables, 6) valores, 7) números, 8) cadenas, 9) caracteres, 10) comentarios. Cuando su documento contiene un elemento a colorear que coincide con múltiples categorías (por ejemplo, palabras clave y comandos), el color de este último en el orden sobrescribirá el de los primeros.

Personalizar el resaltado de sintaxis

Las reglas de coloración de la sintaxis están determinadas por los siguientes ajustes:

CampoDescripción
RESi se selecciona esta opción, tanto la cadena de inicio como la cadena final se considerarán expresiones regulares (RE significa expresión regular).
ICSi se selecciona esta opción, se ignorarán mayúsculas y minúsculas durante la búsqueda (IC significa Ignorar Capitalización). Por ejemplo, si establece "abc" como cadena de inicio y está seleccionada IC, "ABC" o "abc" o "Abc" en su documento coincidirán.
Cadena de inicioUna cadena que denota el comienzo del patrón de búsqueda. Esto es necesario para buscar los elementos de código que quieres colorear.
Cadena finalUna cadena que denota el final del patrón de búsqueda. Este campo se puede dejar en blanco.
DescripciónPuedes usar este campo para dejar un comentario sobre el término. La descripción no afecta al resaltado de sintaxis. Este campo se puede dejar en blanco.
  • Se ignorará una definición cuya cadena de inicio esté en blanco.
  • Una definición cuya cadena final esté en blanco y RE esté desactivada también se utilizará para definir la lista de autocompletado.
  • La configuración RE afecta tanto a la cadena de inicio como a la cadena final, lo que significa que deben escribirse en la expresión regular o en forma simple, ambos juntos.
