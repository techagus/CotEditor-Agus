Asignar archivos a sintaxis en CotEditor

Al abrir un archivo, CotEditor detecta el tipo de documento y aplica la sintaxis correspondiente automáticamente.

Las definiciones de asignación de archivos en cada sintaxis se utilizan para esta detección.

Orden de prioridad de la asignación de archivos

CotEditor detecta la sintaxis correspondiente en el siguiente orden:

  1. Extensión de nombre de archivo
    • Intenta determinar la sintaxis de la extensión del nombre de archivo. Mayúsculas y minúsculas son básicamente ignoradas; sin embargo, los que coinciden con la capitalización se priorizan si se encuentran múltiples sintaxis
    • ej.: .html → HTML
    • ej.: .hTmL → HTML
    • ej.: .py → Python
  2. Nombre del archivo
    • Cuando no hay una sintaxis en la que se defina la extensión de nombre de archivo de búsqueda o el nombre de archivo no tenga una extensión, intenta determinar la sintaxis utilizando el nombre de archivo.
    • ej: .htaccess → Apache
    • ej: makefile → Makefile
  3. Shebang en la primera línea del documento
    • Cuando no hay una sintaxis correspondiente para la extensión y el nombre de archivo, y si la primera línea del contenido del documento es un 'shebang', intenta determinar la sintaxis del shebang. En esta prueba, se utilizan los nombres de los intérpretes que se definen en las sintaxis. El intérprete puede ser la última parte de la ruta justo después de #! o el siguiente término a la ruta si la ruta apunta a env.
    • ej.: #!/usr/bin/perl →Perl (intérprete: perl)
    • ej.: #!/usr/bin/env python3 → Python (intérprete: python3)
  4. Declaración XML
    • Cuando no hay una sintaxis correspondiente hasta aquí, pero el contenido del archivo comienza con una declaración XML, la sintaxis se establece automáticamente en "XML".
    • ej.: <?xml version="1.0" encoding="utf-8"?> → XML
  5. Ninguna
    • Si no se puede encontrar ninguna sintaxis adecuada, se establecería la sintaxis "Ninguna".

Cuando hay varias sintaxis en el mismo nivel, como el nivel de extensión de nombre de archivo o el nivel de nombre de archivo, las sintaxis personalizadas se priorizan sobre las sintaxis incluidas. Y, sin embargo, si hay varias sintaxis correspondientes, la primera sintaxis encontrada se aplicará automáticamente.

Conflictos de mapeo de archivos

No hay conflictos entre las sintaxis incluidas. Sin embargo, si agregó sintaxis personalizadas o modificó las sintaxis incluidas, la asignación de archivos puede entrar en conflicto. Puede ver la existencia de conflictos de asignación desde el comando 'Mostrar conflicto de mapeo de archivos' en el panel Formato de los Ajustes. Si el elemento del menú está en gris, significa que no hay conflicto.

Puedes dejar los conflictos de mapeo de archivos siempre y cuando puedas abrir los archivos con la sintaxis esperada. No hay ningún problema secundario.
